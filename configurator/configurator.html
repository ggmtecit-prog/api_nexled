Tecit Code Logic

Tecit Code Logic

Live Tecit references use a fixed 17-character structure. Each segment maps to one manufacturing field in the configurator.

Reference Pattern 11 | 0375 | 81 | 1 | 1 | 0 | 01 | 01 | 00
Segment Legend
11 Family (2 chars)
0375 Size (4 chars)
81 Color (2 chars)
1 CRI (1 char)
1 Series (1 char)
0 Lens (1 char)
01 Finish (2 chars)
01 Cap (2 chars)
00 Option (2 chars)
Luminos Identity Check

First 10 characters (family + size + color + CRI + series) form identity checked against the Luminos view.

Where Datasheet Data Comes From
Tecit code and Luminos validation

The first 10 characters validate family, size, color, CRI, and series in Luminos. That lookup also defines the product ID used by the API.

Product database

Technical characteristics, electrical data, dimensions, and product descriptions come from the product database used by the NexLed API.

Current PDF assets

Product images, technical drawings, color spectrum files, lens diagrams, icons, and accessory graphics still come from the current local PDF asset library. DAM assets are not part of this PDF flow yet.

Configurator selections

Language, company logo, fixing, power supply, cable, purpose, and other optional fields come from the current configurator selections before the API builds the PDF.
