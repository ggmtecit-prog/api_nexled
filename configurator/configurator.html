    Tecit Code Logic

    Reference Structure

    A Tecit reference is a fixed 17-character code. Each segment maps to one configurator field.

    Length

    17 characters. Longer values are not valid live references.

    Reference Pattern 11 | 0375 | 81 | 1 | 1 | 0 | 01 | 01 | 00
    11
    Family (2 chars)
    0375
    Size (4 chars)
    81
    Color (2 chars)
    1
    CRI (1 char)
    1
    Series (1 char)
    0
    Lens (1 char)
    01
    Finish (2 chars)
    01
    Cap (2 chars)
    00
    Option (2 chars)
    Barra sample
    11037581110010100
    Barra sample with lens
    11007502111010100
    Downlight sample
    29012022191010000
    Dynamic sample
    48007002110010000

    These are the 2-character family prefixes used at the start of a Tecit code.

    Product database
    Technical characteristics, electrical data, dimensions, and product descriptions come from the product database used by the NexLed API.
    Current PDF assets
    Product images, technical drawings, color spectrum files, lens diagrams, icons, and accessory graphics still come from the current local PDF asset library. DAM assets are not part of this PDF flow yet.
    Configurator selections
    Language, company logo, fixing, power supply, cable, purpose, and other optional fields come from the current configurator selections before the API builds the PDF.
